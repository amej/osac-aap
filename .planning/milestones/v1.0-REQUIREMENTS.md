## v1.0 Requirements

### VM Provisioning (PROV)

Core infrastructure provisioning capabilities — creating Windows VMs with correct configuration.

- [x] **PROV-01**: User can boot a Windows VM from an OCI container image using DataVolume registry source — *Validated in Phase 1*
- [x] **PROV-02**: User can specify CPU, memory, and disk sizing via ComputeInstance spec and have it applied to the VirtualMachine — *Validated in Phase 1*
- [x] **PROV-03**: User can connect the Windows VM to a specified VirtualNetwork and Subnet — *Validated in Phase 1*
- [x] **PROV-04**: User can set the Windows hostname via ComputeInstance metadata — *Validated in Phase 1 (15-char NetBIOS truncation)*
- [x] **PROV-05**: System creates VirtualMachine CR with Windows-optimized configuration (virtio-win drivers, Hyper-V enlightenments) — *Validated in Phase 1 (10 Hyper-V features, Windows clock config)*

### VM Verification (VERIFY)

Post-provisioning validation capabilities — confirming Windows VMs are running and accessible.

- [x] **VERIFY-01**: System waits for Windows VM to reach Running state (VirtualMachine.status.ready = True) — *Validated in Phase 1 (create_wait_annotate.yaml, 900s timeout)*
- [ ] **VERIFY-02**: System verifies network reachability by pinging the assigned IP address — *Deferred to v2*
- [ ] **VERIFY-03**: System verifies RDP accessibility by checking port 3389 is reachable — *Deferred to v2*
- [ ] **VERIFY-04**: System verifies QEMU guest agent is responding — *Deferred to v2*
- [ ] **VERIFY-05**: System verifies VNC console is accessible — *Deferred to v2*
